GitHub+Hexo搭建技术博客

发表于 2017-04-01 | 分类于 Hexo | | 阅读次数:

摘要

本文将介绍如何通过git 和hexo 结合搭建一个纯静态的博客网站，会详细介绍git与hexo安装与配置过程，阅读本网您将可以搭建起原生的、基础的基于hexo的博客网站。
快快开始您的博客之旅吧。

一、GitHub+Hexo搭建静态网站原理

  github 作为一个免费开源的代码托管工具，为广大用户提供了纯静态HTML 文件及其网站的解析，所以我们只需要写一个html文件放到github上，即可访问。
  Hexo 刚好具备了可以将动态ejs文件 批量生成HTML文件及其网站的功能，与github不谋而合。
  故，开发者在本地安装Hexo 环境后，批量生成HTML 网站，再将静态网站 通过git命令推送到github上，一个简易的博客网站就算建成了。

二、环境安装

 Hexo 搭建静态博客依赖于2个环境：Hexo NodeJS github

github

1、在github注册账号，并且新建一个仓库，用于存放静态网站，仓库名称请用yourname.github.io,其中(yourname为您的github账户用户名)
2、安装Git客户端，您可以去Git下载，也可以点击博主个人云盘下载。密码：l2ox
3、安装TortoiseGit(可选)，此处可根据个人喜好，选择安装或者不安装，主要提供git的可视化操作，提高效率

1、安装NodeJs客户端，你可以去NodeJs下载，也可以点击博主个人云盘下载。密码：dyi7

四、Hexo静态网站部署

 hexo静态网站部署方式分 本地部署 和 远程部署 ，其中远程部署的方法有很多种，这里我们只讲通过 hexo 快捷部署静态网站

hexo本地部署；

1、运行hexo g 生成静态网页

2、运行hexo s 发布静态网站(发布的网站 home 在blog 的 public 目录)

3、在浏览器中打开网页 http://localhost:4000/ 可验证是否部署成功

hexo远程部署；

1、该插件使用 hexo d 直接可以部署到github上面(需在_congig.xml站点配置文件中配置deploy属性，配置如下，注意空格)

2、站点配置_config.xml配置如下
deploy:
 type: git
 repo:
 github: git@github.com:yourname/yourname.github.io.git,master
3、可登录github 检验代码是否部署到自己所设定的仓库

五、Hexo静态网站备份/多终端同步

  hexo一般情况下，hexo网站部署到github后都是编译过后的静态文件，那假如我们换电脑，或者多终端同步操作，那么这时候我们曾经修改过的配置该怎么同步到现有的及其呢，这里提供一个比较便捷解决方案，再重新新建一个仓库存放我们的数据源信息。
  那么备份哪些文件，成为许多新手头疼的问题，此时大家不用着急，可以执行 hexo clean命令清理工作空间后，在 .gitignore 配置git 在比较和更新时 忽略一下两个文件即可，更多用法，请关注 hexo 简介及其原理


  注意：源文件的备份就只是纯粹的git 操作了，类似于svn ，想了解更多 Git 操作，请多关注 Git章节的内容。
